配置 SSH Key

  • 打开 Git Bash 终端

  • 设置 user.name 和 user.email

    git config --global user.name "你的GitHub用户名"
git config --global user.email "你的GitHub注册邮箱"

  • 生成 SSH Key

    ssh-keygen -t rsa -C "你的GitHub注册邮箱"

    此时，在用户文件夹下就会有一个新的文件夹 .ssh，里面有刚刚创建的ssh密钥文件 id_rsa 和 id_rsa.pub

  • 获取 SSH Key

    cd ~/.ssh
cat id_rsa.pub

    如下图


  • 在 GitHub 中添加 SSH Key

    • 打开 GitHub 网站，点击头像，选择 Settings
    • 选择 SSH and GPG keys，点击 New SSH key
    • 将之前获取到的 id_rsa.pub 的内容复制粘贴到 Key 文本框中

  • 测试是否成功配置 SSH Key

    ssh -T git@github.com
//运行结果出现类似如下
Hi lyfdnrjbrs! You've successfully authenticated, but GitHub does not provide shell access.

    如下图


Hexo

  • 安装 Hexo 的非常简单, 只要一条命令, 前提是安装好 Node.js 与 Git
  • 打开 Git Bash 终端，输入命令 
    npm install -g hexo-cli

搭建博客

初始化 Hexo

打开 Git Bash 终端，输入命令

  • 初始化博客文件夹（我的是在 D 盘创建的 blog，这里可能时间会长些，要耐心等待）

    hexo init /d/blog

    如下图


    初始化成功后生成的一些文件


  • 切换到博客文件夹

    cd /d/blog

  • 安装插件

    npm install

  • 安装其它插件

    npm install hexo-server --save
npm install hexo-admin --save
npm install hexo-generator-archive --save
npm install hexo-generator-feed --save
npm install hexo-generator-search --save
npm install hexo-generator-tag --save
npm install hexo-deployer-git --save
npm install hexo-generator-sitemap --save

本地浏览博客

第一次使用 hexo，建议先在本地创建服务器使用

  • 继续运行 Git Bash，输入命令

    //生成静态页面，可简写为 hexo g
hexo generate
//开启本地服务器，可简写为 hexo s
hexo server

    运行成功后会发现提示信息：


    这里有更多 hexo 常用命令

  • 打开浏览器，地址栏中输入：http://localhost:4000 ，可以看见刚刚创建的博客

    可能出现的问题：访问 http://localhost:4000 无反应？

    解决方法：可能是由于端口问题引起的。使用 Ctrl+C 中断本地服务，使用命令 hexo s -p 5000 重新开启本地服务，访问 http://localhost:5000 就可以看到博客页面了。

    效果图


部署到Github上

  • 打开 blog 根目录下的 _config.yml 文件，拖动到底部找到 deploy 配置，按照以下格式修改并保存： 
    # Deployment
## Docs: https://hexo.io/docs/one-command-deployment
deploy:
  type: git
  repo: git@github.com:lyfdnrjbrs/lyfdnrjbrs.github.io.git
    • 注意：type: git 中的冒号后面有空格
    • 注意：将 repo: git@github.com:lyfdnrjbrs/lyfdnrjbrs.github.io.git 中的用户名换成自己的用户名

  • 打开 Git Bash 终端，输入命令

    //清空静态页面
hexo clean
//生成静态页面
hexo generate
//部署 
hexo deploy

    以上代码可以简写为

    //清除、生成、部署
hexo clean && hexo g -d

    • 注意：第一次发布会提示输入 Github 账号和密码，稍微等待一会，会提示发布成功


  • 打开浏览器，地址栏中输入：http://你的用户名.github.io。如，我的博客地址是：http://lyfdnrjbrs.github.io


NexT 主题

安装 NexT

  • 打开 Git Bash 终端，输入命令

    git clone https://github.com/theme-next/hexo-theme-next themes/next

  • 打开 blog 根目录下的 _config.yml 文件，拖动到底部找到 theme 配置，按照以下格式修改并保存：

    # Extensions
## Plugins: https://hexo.io/plugins/
## Themes: https://hexo.io/themes/
theme: next

配置 NexT

在 Hexo 中有两份主要的配置文件，其名称一样。为了描述方便，在以下说明中，分别用：

站点配置文件 /blog/_config.yml
主题配置文件 /blog/themes/next/_config.yml

区分两个配置文件

Site 设置

打开站点配置文件，找到 # Site，按照以下说明修改并保存：

title: 标题
subtitle: 副标题
description: 简介
keywords: 关键词
author: 作者
language: 语言，zh-CN是简体中文，en是英语
timezone: 时区，默认本地时区

  • 示例

    # Site
title: 标题——了了的个人博客
subtitle: 副标题——学习这件事，不是缺乏时间，而是缺乏努力。
description: 简介——我们可以慢慢向前走，但绝对不能后退。
keywords: lyf
author: 作者——了了
language: zh-CN
timezone: ''

  • 效果图

菜单栏设置

width: 设置侧边栏宽度，默认即可
display: 设置侧边栏显示的时机，默认 post 是在文章页面（拥有目录列表）时显示，always 是在所有页面中都显示，hide 是在所有页面中都隐藏（可以手动展开），remove 是完全移除

  • 示例

  position: right
  #width: 300
  display: post
  padding: 18
  offset: 12

文章目录设置

打开站点配置文件，找到 toc，按照以下说明修改并保存：

enable: 是否启用侧边栏文章目录，默认 true 是启用，false 是不启用
number: 设置是否自动将列表编号，默认 true 是自动编号，false 是不自动编号
wrap: 设置是否换行，默认 false 是不换行，超过长度显示为…，true 是超过长度自动换行
expand_all: 设置是否显示帖子中的所有级别，默认 false 是不显示全部级别，只显示激活部分，true 是将显示帖子中的所有级别

序言文本设置

打开站点配置文件，找到 # Post Settings，按照以下说明修改并保存：

item_text: 是否在序言中显示描述文本，默认 true 是显示，false 是不显示
created_at: 是否在序言中显示帖子创建日期，默认 true 是显示，false 是不显示
updated_at: 的 enable: 是否在序言中显示帖子更新日期，默认 true 是显示，false 是不显示，another_day: 如果更新/编辑的日期与创建日期不同，是否还显示编辑的日期，默认 true 是显示，false 是不显示
categories: 是否在序言中显示分类，默认 false 是不显示全部级别，只显示激活部分，true 是将显示帖子中的所有级别

separated_meta: 是否在序言和脚部部分中显示本文字数和阅读时长，默认 true 是显示，false 是不显示
item_text_post: 是否在序言中显示本文字数和阅读时长的文本描述，默认 true 是显示，false 是不显示
item_text_total: 的 enable: 是否在脚部部分中显示本文字数和阅读时长的文本描述，默认 false 是不显示，true 是显示

注意：要提前在站点根目录执行以下命令进行安装：
示例

npm install hexo-word-counter

tag_icon: 默认情况下，帖子底部的标签在左侧有一个符号 # ，设置是否启用图标代替符号，默认 false 是不显示，true 是显示

  • 示例

    post_meta:
  item_text: true
  created_at: true
  updated_at:
    enable: true
    another_day: true
  categories: true
symbols_count_time:
  separated_meta: true
  item_text_post: true
  item_text_total: true
tag_icon: false
